# Autotrace detector: design, prototype, and measured separation

Structural classifier that flags SVGs machine-traced from raster
(potrace / autotrace / vtracer / Adobe Image Trace) rather than born-vector.
It is the gate that must exist before Wikimedia Commons (5.23M SVGs) can be
mined: Commons' most valuable buckets are also its most Image-Trace-polluted,
and traced ground truth is poison for a cleanup UNet trained to restore *clean*
geometry. The same blind spot killed FIGR-8-SVG, which passed the existing
corpus gate (`gate2`) 100% clean.

Prototype:

- `src/vecml/gate/features.py` — feature extractor (stdlib ElementTree + regex,
  no new dependency; the path tokenizer is a hardened extension of gate2's).
- `scripts/autotrace_detect.py` — builds a labelled set, measures per-feature
  separation, fits a logistic classifier, and runs the svgo adversarial check.
- `docs/research/autotrace-eval/` — `features.json` (per-file feature rows) and
  `model.json` (fitted weights + operating threshold), regenerated by the script.

Reproduce:

```
uv run python scripts/autotrace_detect.py --n 300 --svgo --out docs/research/autotrace-eval
```

## Labelled data

There are no local FIGR samples, so positives are **self-minted**, which is
cleaner than borrowing FIGR anyway: each positive is a trace of a *specific*
born-vector negative, so the two classes are matched on subject matter and any
separation is provenance rather than content. For 300 source SVGs sampled from
`data/audit-500-src` (the svg-stack clean tier):

- NEGATIVE (born-vector): the source SVG itself — 300 files.
- POSITIVE (traced): render each source to a 384px raster, then re-trace with
  **vtracer** (colour) and **potrace** (silhouette) — ~600 files.
- ADVERSARIAL: every minted trace additionally passed through **svgo** with its
  default plugin set — ~600 files.

Caveat carried into every number below: the svg-stack clean tier is not
provenance-filtered (fuel.md rates it medium autotrace-risk), so an unknown
fraction of the "born-vector" negatives are themselves traced. That can only
*understate* the true separation, so the measured figures are conservative.

## Features and per-feature separation

Twenty features, all computed in a single pass over parsed path data. The tells
group into three families: contour geometry (a raster boundary walk produces
long chains of short segments turning by small angles), primitive/topology
absence (tracers emit only filled cubic-Bezier paths — never `<rect>`/`<circle>`
/`<line>`, elliptical arcs, or stroked centre-lines), and coordinate precision
(tracers emit full-precision floats; humans and design tools emit round,
grid-snapped numbers).

Born vs raw-trace class means, sorted by Cohen's d (n=300 born, ~600 trace):

| feature | born mean | trace mean | Cohen's d |
|---|---|---|---|
| `frac_small_turn` (turns < 10 deg between segments) | 0.060 | 0.354 | **1.62** |
| `mean_turn` (mean turning angle, rad) | 1.164 | 0.639 | **1.38** |
| `axis_frac` (axis-aligned share of line segments) | 0.560 | 0.165 | **1.15** |
| `curve_frac` (curve cmds / curve+line cmds) | 0.499 | 0.770 | 0.86 |
| `mean_decimals` (fractional digits per coordinate) | 1.976 | 5.073 | 0.76 |
| `chord_cv` (coeff. of variation of segment length) | 1.182 | 1.694 | 0.72 |
| `has_arc` (any elliptical-arc command) | 0.183 | 0.000 | 0.67 |
| `has_named_id` | 0.170 | 0.000 | 0.64 |
| `prim_frac` (primitive shapes / drawables) | 0.113 | 0.000 | 0.64 |
| `has_stroke` | 0.133 | 0.000 | 0.55 |
| `distinct_fills` | 1.137 | 2.817 | 0.42 |

The contour-geometry family separates hardest. `frac_small_turn` is the single
strongest tell: born-vector art turns sharply at deliberate corners, whereas a
traced contour is a smooth curve chopped into many pieces that each turn only a
few degrees. `axis_frac` captures the born-vector habit of horizontal/vertical
edges (`H`/`V` commands, rectangles) that a boundary walk almost never produces.
Coordinate precision (`mean_decimals`) separates cleanly on raw traces but is
the first tell svgo destroys — see the adversarial section.

Single-feature threshold AUCs confirm the ranking:

| feature | AUC |
|---|---|
| `frac_small_turn` | 0.864 |
| `mean_turn` | 0.846 |
| `axis_frac` | 0.784 |
| `curve_frac` | 0.763 |
| `mean_decimals` | 0.515 |

## Classifier

Logistic regression over the standardized 20-feature vector (implemented in
numpy; no sklearn dependency). Train/test split is **by source file**, so a
trace, its potrace sibling, and its svgo variant never straddle the split —
without that, near-identical rows leak and inflate the AUC.

**Baseline model (trained on born + raw trace):**

- Held-out ROC-AUC: **0.988**
- Operating point at <1% poison admission: keep files with p < 0.519 →
  **95.6% of clean files kept, 0.00% of traces admitted.**

This is the headline separation on un-post-processed traces, and it is already
enough to gate the large fraction of Commons and svg-stack that is raw tracer
output.

## Adversarial: svgo post-processing

svgo's default pipeline rounds coordinates and simplifies paths, which is
precisely an attack on the precision and fine-geometry tells. Measured against
the baseline model:

- **svgo halves the baseline detector.** Of 180 held-out svgo-optimized traces,
  only **47.8% are still caught** at the baseline threshold (mean p 0.47, down
  from ~0 false-negatives on raw traces). This confirms the fuel.md warning that
  structural detection degrades after post-trace editing.

The fix is to train on the attack. svgo cannot restore what tracing destroyed —
it cannot re-introduce `<rect>`/`<circle>`/arcs, strokes, named ids, or the
sharp-corner turning-angle distribution — so a model that has seen svgo output
recovers by leaning on those svgo-invariant structural tells:

**Augmented model (trained on born + raw trace + svgo trace):**

- Held-out ROC-AUC across all positive types: **0.972**
- **98.3% of held-out svgo-optimized traces caught** (mean p 0.90).
- Operating point at <1% poison admission: keep p < 0.396 →
  **75.6% of clean files kept, 0.83% of traces admitted.**

The svgo-robustness costs clean keep-rate (95.6% → 75.6% at the <1% poison
constraint), which is the correct trade for this application: discarding a
quarter of genuine born-vector files is acceptable; letting traced poison into
the ground truth is not. The augmented model's heaviest weights fall on
`frac_lowprec`, `frac_small_turn`, `prim_frac`, `distinct_fills`, `has_stroke`,
and `has_named_id` — the structural core that survives optimization.

**Recommended operating point:** deploy the augmented model at p ≥ 0.40 =
reject (traced). Keeps ~76% of genuine born-vector files while admitting <1% of
traces including svgo-optimized ones.

## Integration into the corpus gate

The detector slots into the existing three-tier `gate2` as a new `autotrace`
reject reason, and **replaces gate2's `trace-suspect` heuristic** (line 236-241
of `scripts/gate2.py`), which only fires above 800 path commands and therefore
missed every short FIGR contour. Placement in the gate2 pipeline:

1. Existing hard rejects first (parse-error, raster, external-ref, zero-size) —
   unchanged; no point scoring an unparseable or raster file.
2. **New autotrace stage:** run `extract_features` on survivors, score with the
   augmented `model.json`, reject when p ≥ 0.40.
3. Existing `warn` tier (live-text, editor-junk) — unchanged.

The feature extractor already mirrors gate2's parser and drawable set, so the
two share idioms and the merge is mechanical. `model.json` ships the weights,
standardization stats, and threshold; scoring is a dot product.

By construction this catches the FIGR fingerprint that defeated gate2: FIGR is
single-`<path>`, no fill, no stroke, dense contour on a uniform 200×200 canvas
— i.e. `path_frac`≈1, `prim_frac`=0, `has_arc`=0, `has_stroke`=0,
`distinct_fills`≤1, high `frac_small_turn`, which is the centre of the trace
cluster. Validating against real FIGR rows is the first next step below.

## Runtime over 5.23M files

Feature extraction measured at **4,442 files/s single-core** on real web SVGs
(svg-stack, median 3.7 KB / p90 11 KB). Scoring is a 20-dimensional dot product,
negligible on top.

- 5.23M files, 1 core: **~20 minutes**
- 5.23M files, 8 cores: **~2.5 minutes**

Detection is not the bottleneck at Commons scale — fetching 5.23M files is
(no media dumps since 2013; category traversal or a polite crawl). The detector
is cheap enough to run on the full pool with no sampling.

## Risks and next steps

- **No real FIGR / Commons validation yet.** Positives are self-minted vtracer
  /potrace/svgo output. The fingerprints should transfer (same tracers underlie
  most Commons Image-Trace uploads), but confirm against ~200 real FIGR-SVG rows
  and ~200 Commons SVGs (some `Created with Potrace`-tagged, some not) before
  trusting the operating point in production. First and cheapest next step.
- **Negative-set contamination.** The svg-stack "born-vector" negatives are not
  provenance-filtered; some are likely traced, which understates separation.
  A hand-audited clean negative set (fonts, flags, Openclipart-minus-autotrace)
  would tighten the numbers.
- **Stronger adversaries than svgo.** Post-trace editing in Illustrator/Inkscape,
  centerline tracing, and manual node cleanup are not covered. Detection is
  known to break down on these; the correct posture (fuel.md) is precision over
  recall — drop the obvious, accept false negatives.
- **Simple geometric shapes** (a traced circle, a born-vector single rectangle)
  live near the boundary and will misclassify either way; they carry little
  training value, so the cost is low.
- **Threshold is base-rate-naive.** The <1% poison figure is measured on a
  balanced mint. On Commons, where the traced fraction of some categories is
  high, revisit the threshold against the observed prevalence to hold the
  clean-verdict precision.
